# Kraven för utveckling av PocketDemocracy

Detta GitHub repository (ev. förvar, förkortat repo), är ett versioneringssystem för kraven av pocketdemocracy.

# Filstruktur

Ett rmtoo projekt (alla projekt i princip) har krav (requirements, sparas i filer med filändelsen '.req'), ämnen (topics, sparas i filer med filändelsen '.tic') och begränsningar (constraints, sparas i filer med filändelsen '.ctr'). Varje filkategori sparas i sin egna mapp med motsvarande namn. När man kör rmtoo går den igenom all informaion, kollar att den är korrekt och producerar utdatafiler i varierande och valbara format (PDF, HTML dom kravdokument och grafer inbäddade i den eller separat).
